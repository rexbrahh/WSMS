# WSL v0 — Working-State Language

Internal, typed, line-oriented operational notation used by observers and the scheduler. Hosted models receive **rendered capsules**, not raw WSL (unless a future mode opts in).

## Design constraints

1. Human-readable (learnable from this page)
2. Typed stable record family
3. Pointer-rich (events, artifacts, file slices)
4. Validated (lint before apply)
5. Compact without cryptic abbreviations
6. Renderable to ordinary structured text

## Record types

| Type | Header | Body fields |
|------|--------|-------------|
| `@task` | `id phase= priority=` | `goal`, `branch?`, `commit?`, `dirty?` |
| `@constraint` | `id hard\|soft source=` | `text`, `scope` |
| `@failure` | `id` | `cmd`, `exit`, `err`, `file_hint?`, `raw?` |
| `@decision` | `id` | `chosen`, `because?`, `refs?`, `scope?` |
| `@avoid` | `id reason=` | `text`, `ref` |
| `@assumption` | `id status=` | `text`, `evidence?` |
| `@invalidated` | `id` | `target`, `reason` |
| `@next` | (no id) | `action`, `target`, `question?` |
| `@page` | `id kind= tier=` | `summary`, `refs?`, `scope?`, `branch?` |
| `@fault` | (optional id) | `kind`, `target` |

## Syntax

```text
@type <id> key=value key=value
field: value
field: "quoted value"
field: `command with spaces`

@type ...
```

- Records start with `@` on a new line.
- Blank line separates records (canonical form).
- Header attrs are `key=value` tokens after the type (and optional id).
- Body lines are `key: value`. Values may be bare, double-quoted, or backtick-quoted.
- ID prefixes: `T` task, `C` constraint, `F` failure, `D` decision, `A` avoid,
  `I` invalidation audit record, `P` page, `E` event.

## Lint rules

1. No active `@decision` may contradict a hard `@constraint` without explicit override.
2. Branch-scoped pages must not auto-promote across branches.
3. `@avoid` and `@invalidated` must reference existing records or events.
4. Exact-evidence fields (`cmd`, `err`, constraint `text`) are immutable once set.
5. Pages whose refs are all stale cannot auto-promote without revalidation.

Scope epochs, stale revisions, terminal status, and path bindings are
replay-derived sidecar metadata rather than new WSL v0 fields. This preserves
canonical WSL compatibility while `internal/coherence` gates L1-L3 residency.
An `@invalidated` record is a provenance-linked audit mapping; it does not erase
its target or replace the authoritative sidecar status.

## Example

```text
@task T42 phase=debugging priority=hot
goal: fix(stream_cancel_hang)
branch: solver-stream-cancel
dirty: src/runtime/stream.go, src/solver/cancel.rs

@constraint C7 hard source=user
text: "do not rewrite transport layer"
scope: task

@failure F18
cmd: `go test ./runtime -run TestCancelStream`
exit: 1
err: "stream goroutine still blocked"
file_hint: src/runtime/stream.go:118-176
raw: artifact:sha256:deadbeef

@avoid A4 reason=failed_attempt
text: "previous goroutine cleanup patch"
ref: F16

@next
action: inspect
target: src/runtime/stream.go:118-176
question: "is cancellation propagated before stream recv blocks?"
```

## Round-trip invariant

`Parse(Serialize(records))` is semantically equal to `records`. The serializer defines canonical field order.

## L3 indexing boundary

WSL v0 page IDs and exact refs are logical addresses; vector embeddings and
search-backend metadata are not WSL syntax. The post-demo L3 page compiler may
derive richer searchable metadata, versions, source digests, access statistics,
and embedding namespaces beside a `@page`, but those fields remain disposable
index state under `docs/l3-warm-memory.md`. A vector hit must resolve back to a
current WSL/L4 address before it can be rendered.
